# AMPLIFY — 用結構和計算買回能力

> 讀者：弱模型本人，以及編排它的人。
> 誠實前提：**沒有方法讓弱模型單發輸出達到強模型水準。**
> 但單發不是唯一玩法 — 弱模型多跑幾次＋外部驗證＋分工，
> 在「可驗證的任務」上能買回大部分差距。本檔是那些玩法的操作手冊。

## 先分清楚：什麼買得回、什麼買不回

| 能力 | 買得回嗎 | 用什麼買 |
|---|---|---|
| 產出品質（可驗證任務：code、資料、格式轉換） | **大部分** | 放大器 A–C |
| 不遺漏約束 | **大部分** | LONGRUN 外部化 + 放大器 B |
| 校準（知道自己哪裡錯） | 部分 | 放大器 B（換一個實例當批評者） |
| 品味（沒有判準的好壞） | 少量 | 放大器 E（黃金範例） |
| 原創洞見、跨域連結 | **買不回** | 升級（LIMITS 階梯）或接受 |

任務屬於最後一列 → 別浪費計算硬跑，直接走 signals/LIMITS.md 升級階梯。

## 放大器 A — 先造判準，再開工（最重要的一個）

- **原理**：弱模型「生成對的東西」很難，「判斷東西對不對」容易得多。所以把每個任務盡量轉成有外部判準的形式。
- **程序**：動工前先問「做完之後，**什麼機械性檢查能證明它對**？」→ 先把那個檢查造出來（測試、對帳腳本、驗收清單、範例輸入輸出對），才開始做。
- 例：「寫一個轉換程式」→ 先寫 5 組輸入輸出對；「整理報告」→ 先列驗收清單。
- 造不出任何判準的任務 = 品味任務 → 放大器 E 或升級。

## 放大器 B — 生成 → 批判 → 修訂（換一個腦批判）

- **程序**：
  1. 正常完成產出。
  2. 把產出交給**另一個實例 / 另一個模型**（新對話，不帶你的推理過程），用中立偏對抗的問題：「找出這份東西的 3 個問題，按嚴重度排序。」— 禁止問「這樣對嗎？」（會誘導同意）。
  3. 對每條批評：先驗證批評本身是否成立（批評者也會幻覺），成立才修。
  4. 高風險任務跑 2 輪；批評第 2 輪找不到新問題 → 收工。
- **為什麼要換腦**：同一個 context 裡自我批判，會被自己的推理路徑污染 — 你會沿著同樣的盲點再走一次。乾淨的 context 是批判價值的來源。

## 放大器 C — N 選 1（短而重要的產出）

- **程序**：同一任務獨立做 3 次（每次乾淨 context）→ 比較：
  - 三次答案一致 → 信心高，採用。
  - 不一致 → **不一致點就是任務的困難點**，把力氣集中在那裡（查證、升級、或呈給使用者）。
- 適用：關鍵決定、數字計算、事實判斷、重要措辭。不適用：長產出（比較成本太高，改用 B）。

## 放大器 D — 強弱分工（有多個模型等級可用時）

- **預算規則**：強模型的 token 花在**兩端** — 開頭的計畫和結尾的審查；中間的執行給弱模型。
- 開頭：強模型把任務切成「每塊都可獨立驗證」的計畫（存進 WORKING.md 格式）。
- 中間：弱模型逐塊執行，每塊跑 VERIFY 配方。
- 結尾：強模型只看 diff / 成品 + 驗證輸出，做最終審查。
- 沒有強模型可用 → 計畫和審查兩端改用放大器 B（換實例）+ C（計畫做 3 版選 1）。

## 放大器 E — 黃金範例錨定（品味任務的唯一解）

- **原理**：弱模型無法從規則長出品味，但**模仿一個好範例的形狀**能達到七成。
- **程序**：做任務類型 X 之前，找一個公認高品質的 X 實例（好的 README、好的報告、好的 commit 訊息），放進 context，指令：「形狀、密度、語氣照這個範例，內容換成本任務。」
- **建置**：遇到使用者稱讚過的產出 → 存進 `gold/` 目錄（`gold/report.md`、`gold/commit.txt`…）。這個目錄的價值隨時間累積，是品味的外部記憶體。

## 餵弱模型的方式（給編排者：人或上層 agent）

1. **一個 session 一個任務。** 弱模型混多任務必掉約束。
2. **關鍵約束放 prompt 最後**，並要求它先複述約束再動工。
3. **貼原文，不要轉述。** 你的轉述會丟掉弱模型需要的細節——逐字原則對餵入方向同樣成立。
4. **小 context 贏過大 context。** 給它需要的 3 個檔案，不是整個 repo — 弱模型在長 context 中間的資訊會讀丟。
5. 要求結構化輸出（模板、清單、JSON）— 結構是弱模型的扶手。

## 成本對照（別對 5 分鐘任務開放大器）

| 任務風險（J4 分級） | 用什麼 |
|---|---|
| 重做 5 分鐘級 | 單發，不放大 |
| 重做 1 小時級 | A（先造判準）+ 一輪 B |
| 無法挽回級 | A + B×2 + C，或 D 的強模型審查端 |
