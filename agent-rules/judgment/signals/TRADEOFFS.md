# TRADEOFFS — 深度利益權衡（CHOICES 的深水區）

> 讀者：任何模型。觸發時機：signals/CHOICES.md 的 5 分鐘表決不了、或選擇是單向門、
> 或涉及大金額、或影響使用者以外的人——任一成立就升級到本檔。
> 核心：淺權衡比較的是選項的**標籤**（快/慢、貴/便宜）；深權衡比較的是
> **每個受影響者在每個時間點的完整帳**。錯的權衡多半不是算錯，是漏了一方或漏了一段時間。

## 程序（依序執行，產出見底部模板）

### 第 1 步 — 利害關係人表：誰得到什麼、誰付出什麼

- **動作**：列出所有受影響者，**特別是不在場的**：維運的人、下一任接手的人、終端使用者、付錢的人。每個選項 × 每個關係人填「得到／付出」。沉默方沒有代表出席 → 由你替他們填，並在交付時標明。
- **自檢**：表裡有至少一個「不在這場對話裡的人」嗎？
- **例**：選 self-host 省授權費 → 表上補「維運同事：付出每月 on-call ＋升級工」——省的錢是別人的加班換的，這行不寫出來就是把成本藏進別人的口袋。

### 第 2 步 — 全成本帳：標價之外的四種錢

- **動作**：每個選項估四項（數量級即可）：**學習成本**（誰要學多久）、**遷移成本**（進去要搬什麼）、**維運成本**（每月人時＋帳單）、**退出成本**（兩年後想離開要付多少——資料搬得走嗎？格式綁死嗎？）。退出成本最常被漏，而它決定你未來的談判地位。
- **自檢**：四項每個選項都有值？「退出成本」那格不是空的？
- **例**：SaaS A 月費比 B 便宜 30% → 但 A 的資料匯出只給 PDF——退出成本近乎無限大，便宜 30% 是鎖鏈的價格。

### 第 3 步 — 兩個時間點各評一次

- **動作**：同一張表在「現在」和「一個相關的未來點」（規模 ×5 時、合約到期時、團隊翻倍時——選跟這個決定壽命相符的）各評一輪。兩輪的贏家不同 → 明寫「短期贏家 A、長期贏家 B」，並回答：我們付得起先選 A 之後再換 B 的轉換成本嗎？付得起 → 選 A 不虧；付不起 → 這其實是單向門，照單向門處理。
- **自檢**：有兩輪評分？翻轉有被處理而不是被忽略？

### 第 4 步 — 來源利益申報

- **動作**：權衡用到的每份輸入資料標注：**誰寫的、他希望你選哪個**。Vendor 的 benchmark、作者是擁護者的比較文、業務給的報價，都不是中立數據——不是不能用，是要打折並找對向資料平衡。
- **自檢**：每份關鍵依據都標了來源立場嗎？有沒有某個選項的資料全部來自它自己的陣營？
- **例**：資料庫選型三份依據全是 A 廠官網 → 補一份 A 的失敗案例檢索再評。

### 第 5 步 — 不可逆性計價

- **動作**：每個選項寫一行「反悔要花 ___」。兩案總分接近（差距 <20%）→ **可逆的那個直接勝出**，不再糾結——可逆性本身就是價值（選錯可以改 = 期望損失砍半）。
- **自檢**：每個選項都有反悔價？接近時讓可逆者贏了嗎？

### 第 6 步 — 隱形項強制列出

- **動作**：把不可量化的因素明列進表：團隊士氣、信任、專注力碎片化、對外名聲、法規風險。每項只標方向（+/−/0）和一句理由。**禁止因為量化不了就當作零**——量化不了的成本照樣要付。
- **自檢**：表裡有非數字列嗎？

### 第 7 步 — 打破給定的選項框

- **動作**：評分前強制回答三問：1. **都不選**會怎樣？（現狀撐多久？——現狀是永遠的隱藏選項）2. 能不能**都要或分階段**？（先 A 後 B、小規模試 B）3. 有沒有**第三選項**是給定框架故意或無意排除的？
- **自檢**：三問都有答案，且「現狀」出現在比較表裡？
- **例**：「選 K8s 還是 ECS」→ 三問揭穿：現狀（docker-compose 單機）其實還能撐 18 個月，答案是「現在都不選，設好觸發條件再回來」。

### 第 8 步 — 權重先於評分

- **動作**：看選項評分**之前**，先寫下判準與權重（哪些是一票否決的硬條件、剩下的相對重要性）。順序反了 = 事後合理化——你會不自覺調權重去支持已經喜歡的那個。
- **自檢**：權重的寫下時間在評分之前嗎？

### 第 9 步 — 敏感度測試

- **動作**：找出整個權衡裡**最不確定的那個假設**（通常在 ASSUMED 堆裡），把它翻轉：贏家會換人嗎？會 → 這個假設就是真正的決策點，先花力氣驗證它，再回來選。不會 → 決策對這個不確定性免疫，放心選。
- **自檢**：跑過至少一次翻轉測試？

### 第 10 步 — 交付：輸家要被明說

- **動作**：用 signals/CHOICES.md 的 4 行模板（DECIDED / BECAUSE / REJECTED / REVISIT-IF），外加兩行：
  - `COSTS-FALL-ON: <這個決定的代價由誰承擔>`
  - `MITIGATION: <對承擔者的補償或至少告知>`
  被犧牲的利益要寫在紙上讓決策者簽收，不是靠沒人發現。
- **自檢**：交付物裡指得出誰是這個決定的輸家嗎？（「沒有輸家」幾乎總是「沒找到輸家」）

## 輸出模板（照填）

```
DECISION: <一句話>
STAKEHOLDERS: <表：誰 × 得到/付出>
FULL COST: <表：選項 × 學習/遷移/維運/退出>
NOW vs LATER: <兩輪贏家；若翻轉，轉換成本評估>
SOURCE BIAS: <關鍵依據的立場標注>
REVERSIBILITY: <各選項反悔價>
UNQUANTIFIED: <方向判斷清單>
SENSITIVITY: <翻轉了哪個假設、結果>
DECIDED / BECAUSE / REJECTED / REVISIT-IF
COSTS-FALL-ON / MITIGATION
```

## 禁語表

| 禁語 | 替代寫法 |
|---|---|
| 「整體來說 A 比較好」 | 「對 ___ 來說 A 好在 ___；代價由 ___ 承擔」 |
| 「B 比較便宜」 | 「B 標價低 30%；四項全成本後差 ___（含退出成本）」 |
| 「這沒辦法量化就先不考慮」 | 列進表，標方向與理由 |
| 「反正大家都選 A」 | 別人的權重不是你的權重；跑第 8 步 |
| 「沒有輸家」 | 找到為止，或寫「輸家：未來的維運者，代價 ___」 |
