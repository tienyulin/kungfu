# DISTILL — 長文提煉規則系統

> 讀者：任何模型（規則以 7B 等級為下限設計——弱模型照字面執行，強模型當檢查清單）。
> 適用：把上萬字的文件、報告、逐字稿、log、討論串，提煉成關鍵結論。
> 每條規則三段：觸發（可觀察）→ 動作（可執行）→ 自檢（yes/no）。
> 優先序：使用者最新明確訊息 > 專案 CLAUDE.md > 本檔 > 你的直覺。本檔沒寫到的情況，用 KERNEL.md。
> 核心：**摘要是替讀者做完閱讀的工作，不是把全文等比例縮小。** 縮小版全文 = 沒做工。

## R1 — 先問「讀者拿這個做什麼決定」

- **觸發**：收到摘要 / 提煉 / 重點整理任務。
- **動作**：寫下一行：「讀者是 ___，讀完要能 ___」。使用者沒說 → 低風險自己判（記 ASSUMED），高風險問一句。這一行決定什麼算重點——同一份財報，給投資人和給稽核的重點是兩份不同清單。
- **自檢**：我的產出開頭想著一個具體讀者和一個具體用途嗎？
- **例**：一份 50 頁事故報告 → 錯：均勻摘要每一章。→ 對：讀者是要決定「要不要改架構」的主管 → 重點是根因、影響範圍、修復成本三件事。

## R2 — 兩遍讀法：先掃載重點，再細讀載重段

- **觸發**：原文超過 3000 字。
- **動作**：第一遍只掃結構找**載重句**的位置：標題、每段首尾句、表格、數字、粗體、「結論／建議／但是／除了／必須」附近。標記位置。第二遍只細讀被標記的段落。禁止從第一個字順讀到最後一個字——那是把時間均勻花在不均勻的價值上。
- **自檢**：我有一份載重點位置清單嗎？
- **例**：120 頁合約 → 掃出金額、期限、終止條款、違約條款、附件清單的位置，細讀這五處；描述性條款掃過即可。

## R3 — 結論先行的金字塔，禁止目錄式摘要

- **觸發**：開始寫產出。
- **動作**：格式固定：1 句 bottom line（讀者只能帶走一句時的那句）→ 3–5 個支撐點（每點附原文位置）→ 需要的細節。**禁止**「本文討論了 X、Y、Z」「文件涵蓋以下主題」——那是目錄，不是結論；讀者要的是文件**說了什麼**，不是文件**裡有什麼**。
- **自檢**：第一句是一個可以被反駁的結論，還是一個主題列表？（主題列表 = 重寫）
- **例**：錯：「報告討論了 Q3 業績、成本結構與明年展望。」→ 對：「Q3 獲利成長全部來自成本削減，營收實際下滑 4%——明年展望建立在營收回升的假設上，但報告沒有給出依據（p.23）。」

## R4 — 數字與限定詞原樣搬運

- **觸發**：原文有數字、或有限定詞（「僅限」「初步」「在 X 條件下」「尚未經審計」）的結論。
- **動作**：1. 數字帶著單位、基期、樣本數走：「顯著改善」→ 貼原數字「+3.2%（n=12）」。2. 限定詞必須跟著它修飾的結論一起出現在摘要裡——去掉「初步」的初步結果是另一個結論。3. 你算出來的衍生數字（成長率、比例）標「推算」。
- **自檢**：摘要裡每個結論的力度，和原文的力度一樣嗎？（更強 = 你在替作者吹牛）
- **例**：原文「初步數據顯示，在實驗室條件下延長 20%」→ 錯：「壽命延長 20%」。→ 對：原句照搬，兩個限定詞都在。

## R5 — 專門跑一遍「找埋雷」

- **觸發**：文件是合約、報告、政策、事故檢討等有利害關係的文本。
- **動作**：獨立跑一遍只找五種東西：**例外條款**（「除了」「但不包括」）、**免責與限制**、**註腳與附錄**、**期限與義務**（日期、金額、罰則）、**與主文調性矛盾的句子**。這些東西的位置故意不顯眼，但對決策的影響常大於整個主文。
- **自檢**：埋雷清單跑過了嗎？找到的雷放進摘要前三點了嗎？
- **例**：40 頁全在講產品多好，p.37 註腳一行「本服務等級不適用於免費方案」——這行就是重點，不是那 40 頁。

## R6 — 主張、證據、轉述，三種分開標

- **觸發**：原文任何陳述要進摘要。
- **動作**：分三類寫：**作者主張**（「作者認為 X」）、**數據顯示**（「數據：X（p.N）」）、**二手轉述**（「作者引用 Y 稱 X」）。作者的主張不因為被你摘要就升級成事實。輸出語氣照 KERNEL J2 分級。
- **自檢**：摘要裡有沒有把「作者說的」寫成「事實是」的句子？
- **例**：錯：「新製程降低成本 30%。」→ 對：「作者宣稱新製程降本 30%，依據是內部試產數據（未附原始資料，p.12）。」

## R7 — 矛盾照實列，不准替作者圓場

- **觸發**：原文內部互相矛盾（前後數字不一致、結論與數據打架）。
- **動作**：兩處都指出來：「p.5 說 A，p.30 說非 A」。禁止自己挑一邊、或用「大致上」平滑掉——**矛盾本身常是全文最有價值的資訊**（代表資料有問題或有人沒對齊）。
- **自檢**：發現的矛盾都進了摘要，還是被我修掉了？

## R8 — 每個結論可回溯

- **觸發**：寫下任何一條摘要結論。
- **動作**：附原文位置（頁碼／章節／時間戳），讓讀者 30 秒內能翻到出處驗證。
- **自檢**：隨機挑一條結論，我能立刻指回原文位置嗎？
- **例**：「營收下滑 4%（表 3，p.8）」。

## R9 — 段落之間的因果，是你的就標你的

- **觸發**：你把原文兩處內容連成一個因果或推論（「因為 A 所以 B」而原文沒這樣說）。
- **動作**：標「推論：」。原文沒說的話不能以原文的名義出現——摘要最危險的幻覺不是編內容，是編**連接**。
- **自檢**：摘要裡每個「因此／所以／導致」，原文有對應句嗎？沒有的標了推論嗎？

## R10 — 長度由重點數量決定，不由原文長度決定

- **觸發**：決定產出多長。
- **動作**：1. 一萬字的文件如果只有三個重點，摘要就是三點，不因原文長而灌水。2. 讀者可能要不同深度 → 分層交付：一句話版 → 一頁版 → 需要時指向原文段落。3. 砍到不能再砍的判準：再刪一句，讀者的決定會不會不同？會 → 留；不會 → 刪。
- **自檢**：摘要裡有沒有「為了顯得完整」而存在的句子？

## R11 — 行動項與期限獨立成節

- **觸發**：原文含任何要人做的事：期限、待決事項、要求、承諾。
- **動作**：抽出來單獨列一節「行動項」：誰、做什麼、期限（絕對日期）。混在敘述裡的行動項等於被藏起來。
- **自檢**：原文裡每個日期和「請／需／應」句，都被掃過一次了嗎？

## 回合終檢（結束前逐題答，任一 no → 先修再結束）

1. 開頭那句是可被反駁的結論，不是主題列表？
2. 數字帶單位基期、限定詞跟著結論走？
3. 埋雷 pass 跑過（例外／免責／註腳／期限／矛盾）？
4. 主張／證據／轉述分開標了？
5. 每條結論可回溯到原文位置？
6. 我加的推論都標了「推論」？

## 禁語表

| 禁語 | 替代寫法 |
|---|---|
| 「本文討論了 X、Y、Z」 | 直接說文件的結論：「X 的核心發現是 ___」 |
| 「顯著提升／大幅改善」 | 貼原數字：「+3.2%（n=12，p.7）」 |
| 「總的來說內容很豐富」 | 摘要裡不評價文件，只提煉它 |
| 「作者證明了 X」 | 「作者主張 X，依據是 ___（p.N）」 |
| 「細節請見原文」（無位置） | 「見 p.N §M」——可回溯才算指路 |
