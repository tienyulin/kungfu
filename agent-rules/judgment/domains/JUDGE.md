# JUDGE — 評判規則系統

> 讀者：任何模型（規則以 7B 等級為下限設計——弱模型照字面執行，強模型當檢查清單）。
> 適用：評判任何東西 — 產出物（code、文件、設計、翻譯）、想法、提案、計畫、別人的評判。
> 每條規則三段：觸發（可觀察）→ 動作（可執行）→ 自檢（yes/no）。
> 優先序：使用者最新明確訊息 > 專案 CLAUDE.md > 本檔 > 你的直覺。
> 核心：評判的敵人不是嚴不嚴，是**偏誤**——第一印象、光環、順序、和稀泥。
> 每條規則都在拆一種偏誤。

## R1 — 先寫 rubric，再看作品

- **觸發**：收到「評一下 / review / 打分 / 這個好不好」類任務。
- **動作**：**看作品之前**先寫下：1. 這東西要替誰達成什麼（用途一行）。2. 評判維度 3–6 個（從用途推導，不是通用形容詞）。3. 每個維度的權重。使用者給了標準 → 用他的。順序反了（先看再定標準）= 標準會被第一印象污染，你會替直覺找理由。
- **自檢**：rubric 的寫下時間在打開作品之前嗎？
- **例**：評一份 onboarding 文件 → 錯：讀完覺得「寫得挺好」再想理由。→ 對：先寫「用途：新人第一天能自己跑起環境。維度：步驟可跟性 50%、前置假設明確性 30%、過期風險 20%」，再打開文件。

## R2 — 致命傷先查，gate 不過不用細評

- **觸發**：rubric 定好，開始評。
- **動作**：先查一票否決項（依對象而定）：抄襲/造假、安全漏洞、不符硬需求、數據錯誤、法律問題。中了 → 直接輸出「gate 不過：___」，不用再花力氣打維度分——**細評一個 gate 不過的東西是在浪費雙方時間，還會用 7 分的細節分軟化一個 0 分的事實**。
- **自檢**：gate 清單查了嗎？查在細評之前嗎？
- **例**：提案數據漂亮 → gate 查發現核心數字來源造假 → 結論就是這一行，不是「數據有疑慮但結構清晰，綜合 6 分」。

## R3 — 分維度獨立打分，最後才看總分

- **觸發**：開始打分。
- **動作**：一個維度一個維度評，每個維度評完才進下一個。禁止先形成整體印象再回填分數——那是光環效應：一個亮點（文筆好）會污染所有維度（連數據都覺得可信了）。總分最後算，算出來跟直覺差很多 → 檢查是直覺錯還是權重錯，寫下是哪個。
- **自檢**：有沒有哪個維度的分數，理由其實來自別的維度？

## R4 — 每個分數附作品裡的證據

- **觸發**：寫下任何一個維度分數。
- **動作**：附上作品中的具體位置（引用原句、行號、頁碼、畫面）。寫不出證據的分數 = 印象分，删掉重評。批評尤其要可指認：「第 3 節的步驟 4 假設讀者已裝好 Docker，但前文沒提」，不是「假設太多」。
- **自檢**：隨機挑一個分數，30 秒內指得出它的證據位置嗎？

## R5 — 優點和缺點分兩遍找，各找到額度為止

- **觸發**：單一 pass 評完，準備下結論。
- **動作**：強制跑兩遍：**找優點遍**（至少 2 個，含「哪些東西改的時候不准弄壞」）和**找缺點遍**（至少 2 個）。第一印象好 → 缺點遍要加倍認真；第一印象差 → 優點遍加倍認真。這是在拆確認偏誤：你的預設行為是只蒐集支持第一印象的證據。
- **自檢**：兩遍都跑了？逆著第一印象的那遍有真實產出？
- **例**：爛提案裡的好想法：整體不可行，但第 4 節的定價思路值得單獨留下——沒跑優點遍就會跟著提案一起被扔掉。

## R6 — 事實錯誤與品味分歧分開標

- **觸發**：想扣分或批評的每一點。
- **動作**：先分類：**事實錯誤**（可驗證：數字錯、code 跑不了、引用不存在）→ 驗證後才寫，附驗證方式。**品味分歧**（我會用另一種寫法/結構/風格）→ 標「品味：」，並回答「作者的選擇有沒有合理理由？」有 → 不扣分，只記備註。把品味當錯誤扣分是最常見的爛評審行為。
- **自檢**：每條批評都有「事實／品味」標籤？事實類都驗過？

## R7 — 比較評判要同條件＋防順序偏誤

- **觸發**：評判對象是兩個以上（A vs B、多方案排名）。
- **動作**：1. 同一份 rubric、同樣的輸入條件評所有對象。2. **先各自獨立評完，才互相比**——先比較會讓第一個看的變成錨點。3. 分數接近時（差 <15%）交換順序再評一次，結果翻轉 = 你評的是順序不是品質，重做。
- **自檢**：獨立評分完成於比較之前？

## R8 — Findings 分級，禁止 20 條平鋪

- **觸發**：問題找完，準備輸出。
- **動作**：每條標級：**blocker**（不修不能用）／**major**（顯著損害目的）／**minor**（小傷）／**nit**（吹毛求疵，可忽略）。輸出按級別排序。10 條 nit 混著 1 條 blocker 平鋪 = 讀者會漏掉那條 blocker——分級是評審對讀者的義務。
- **自檢**：有分級？blocker 在最上面？nit 有標明可忽略？

## R9 — major 以上的批評，附至少一個修法方向

- **觸發**：寫下任何 blocker / major。
- **動作**：附一句「往 ___ 方向修」或「參考 ___ 的做法」。不必替他做完，但**只拆不指路的評判是裁判不是評審**。真的不知道怎麼修 → 寫「我沒有修法建議，但問題成立因為 ___」，誠實比硬掰好。
- **自檢**：每條 major+ 都有方向或誠實聲明？

## R10 — 評自己的產出：換帽子＋隔一個任務

- **觸發**：評判對象是自己（或同 session）產出的東西。
- **動作**：1. 假設它是陌生人交來的，第一句心裡話是什麼？寫下來。2. 可以的話隔一個任務再評（冷卻）。3. 至少跑一遍 signals/FAILMODES.md 症狀表。4. 自評結果全是好話 → 視同沒評，標注「自評，未經獨立檢核」。
- **自檢**：自評有找到至少一個真實缺點嗎？（找不到 = 沒認真評，不是沒缺點）

## R11 — 承認自己不是合格評審的時候

- **觸發**：評判對象的領域，你連「好的長什麼樣」都說不出來（rubric 第 1 步寫不出來）。
- **動作**：照 signals/LIMITS.md：能評的維度照評（邏輯一致性、內部矛盾、可讀性——這些跨領域通用），領域專業維度標「信心低，建議 ___ 領域的人複核」。禁止用通用維度的分數冒充領域判斷。
- **自檢**：我的每個維度分數，我夠格打嗎？

## R12 — 評判輸出格式（照這個順序）

```
VERDICT: <一句話結論＋建議動作（採用/修後採用/退回/淘汰）>
GATE: <一票否決項檢查結果>
| 維度 | 權重 | 分 | 證據 |
KEEP: <改的時候不准弄壞的優點>
FINDINGS: <按 blocker/major/minor/nit 排序，major+ 附修法方向>
CONFIDENCE: <哪些維度我夠格、哪些建議複核>
```

## 回合終檢（結束前逐題答，任一 no → 先修再結束）

1. rubric 寫在看作品之前？
2. gate 查在細評之前？
3. 每個分數有可指認的證據？
4. 逆著第一印象的那遍跑了且有產出？
5. 事實／品味分開，事實類驗證過？
6. findings 有分級、major+ 有方向、KEEP 有內容？

## 禁語表

| 禁語 | 替代寫法 |
|---|---|
| 「整體來說不錯」 | VERDICT 一句話＋維度表 |
| 「有些小問題」 | 列出來、分級、附位置 |
| 「LGTM」（無證據） | 「查了 ___ 和 ___，維度分如下，無 blocker」 |
| 「我覺得這樣寫比較好」 | 「品味：我會 ___；作者的選擇因 ___ 也成立」或升級成有證據的 major |
| 「這不行」（無指路） | 「blocker：___（證據）。修法方向：___」 |
| 「寫得很用心」 | 用心不是維度；說它達成了什麼 |
