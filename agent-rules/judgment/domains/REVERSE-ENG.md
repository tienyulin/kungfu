# REVERSE-ENG — 逆向工程規則系統

> 讀者：任何模型（規則以 7B 等級為下限設計——弱模型照字面執行，強模型當檢查清單）。
> 適用：搞懂沒有文件的東西 — legacy 程式碼、未知資料格式、別人留下的系統、
> 無文件 API、黑箱行為。前提：**對象是自家或已授權的系統**；未授權 → 停，先問。
> 每條規則三段：觸發（可觀察）→ 動作（可執行）→ 自檢（yes/no）。
> 優先序：使用者最新明確訊息 > 專案 CLAUDE.md > 本檔 > 你的直覺。
> 核心：逆向不是「把它全部讀懂」，是**用最少的觀察回答一個具體問題**。

## R1 — 先寫下要回答的問題，全盤理解不是目標

- **觸發**：收到「搞懂這個系統／這段 code／這個格式」類任務。
- **動作**：寫一行：「我要回答的問題是 ___」（例：「訂單狀態在哪裡被改成 cancelled」，不是「理解訂單系統」）。使用者真的要全盤理解 → 拆成問題清單，逐個攻。
- **自檢**：我有一個可以判定「答完了沒」的具體問題嗎？
- **例**：錯：從第一個檔案開始順讀 30 萬行 legacy。→ 對：問題是「為什麼有些單少算運費」→ 只追運費計算路徑。

## R2 — 行為是證據，名字和註解只是線索

- **觸發**：讀到任何命名、註解、舊文件。
- **動作**：把它們當**假設來源**，不當事實：`calculateTax()` 可能早就不只算稅，註解可能是三年前的。要當事實引用的行為，必須實際觀察過（跑過、抓過封包、看過輸出）。文件與行為矛盾 → **信行為**，把矛盾記進地圖（它本身是資訊：文件何時過期的）。
- **自檢**：我的結論裡每個「它會做 X」，是看名字推的還是看行為證的？
- **例**：註解說「此欄位已棄用」→ 實測發現下游報表還在讀它——刪了就炸。行為勝。

## R3 — 從邊界進入：先畫黑箱 I/O，再開箱

- **觸發**：面對一個還沒開始分析的系統。
- **動作**：開箱前先填黑箱表：**什麼會進去**（輸入、觸發、排程）、**什麼會出來**（輸出、副作用、寫了哪些地方）、**入口點在哪**（main、route 表、exported functions、cron）。從入口點和 I/O 邊界往內追，不從檔案樹頂端往下讀。
- **自檢**：我有黑箱 I/O 表了嗎？我目前讀的每段 code 都在某條「入口→出口」路徑上嗎？
- **例**：未知的資料處理服務 → 先看：吃哪個 queue、寫哪張表、暴露哪些 endpoint——三個答案就框出了全部功能範圍。

## R4 — 跟著一筆具體資料走，不是跟著檔案走

- **觸發**：要理解一條處理流程。
- **動作**：挑**一筆真實的具體輸入**（一個訂單、一個請求、一列資料），從進入點開始追它的完整旅程：被誰讀、被怎麼變換、寫到哪。讀 code 的順序由這筆資料的路徑決定。抽象讀不懂的，具體資料一走就懂。
- **自檢**：我現在追的是一筆具體資料，還是在漫遊檔案？
- **例**：搞不懂折扣邏輯 → 造一筆「訂單 #1234，兩件商品，一張折價券」，逐函式追它，比讀完整個 pricing 模組快十倍。

## R5 — 假設→實驗迴圈：沒驗證的理解是 GUESS

- **觸發**：形成任何「我覺得它是這樣運作」的理解。
- **動作**：立刻設計一個**可觀察的實驗**驗證它：改一個輸入看輸出變不變、下斷點看走不走到、加一行 log、發一個測試封包。實驗要有預測——「如果我的理解對，做 X 會看到 Y」。看到 Y → 升級成已驗證；沒看到 → 理解錯了，這是好消息，趁早知道。
- **自檢**：地圖裡每條「已確認」都有對應的實驗紀錄嗎？
- **例**：「這個 cache 應該 5 分鐘過期」→ 寫入一筆，5 分 01 秒後讀——不做這步，你的「理解」只是讀 code 讀出來的猜測。

## R6 — 地圖檔隨手更新，發現不寫下等於沒發現

- **觸發**：任何逆向任務開始時。
- **動作**：建 `MAP.md`，三節：**已確認**（行為＋證據＋實驗方式）、**未解問題**（排優先序）、**死路**（試過不通的，防止重走）。每有發現當下就寫。session 重啟從地圖恢復，不從記憶恢復。
- **自檢**：過去 30 分鐘的發現都在地圖裡嗎？
- **例**：沒地圖的第三天：「這段我好像看過……它做什麼來著」——同一段 code 讀三遍，是逆向工程最大的時間黑洞。

## R7 — 對照實驗定位功能：diff 兩個世界

- **觸發**：想知道某個設定／版本／輸入片段是做什麼的。
- **動作**：準備兩個只差一處的世界（開關 flag、兩個版本、欄位有值/沒值），跑同樣操作，diff 行為。一次只差一處（KERNEL J5），差異就是那一處的功能定義。
- **自檢**：我的兩個對照組真的只差一個變因嗎？
- **例**：不知道設定檔裡 `legacy_mode=true` 幹嘛 → 副本上關掉跑測試流程，diff 出三個行為差異，比讀 code 快且不會漏執行時分支。

## R8 — 探測只在副本上做，活系統只准觀察

- **觸發**：實驗需要改值、下斷點、fuzz 輸入、觸發錯誤路徑。
- **動作**：一律在副本／sandbox／本地環境做。正在服務使用者的系統上只允許**被動觀察**（讀 log、讀 metrics、抓流量）。副本建不起來 → 回報，跟使用者要一個，不要「小心一點直接試」。
- **自檢**：我的每個主動實驗，跑在哪裡？
- **例**：想知道「送壞格式會怎樣」→ 在 prod 試 = 拿真使用者當實驗品；副本上 fuzz 到爽都沒關係。

## R9 — 一次觀察不算行為，至少三種輸入才准說「它會」

- **觸發**：想把一次觀察寫成通則（「它會把空值轉成 0」）。
- **動作**：至少換三種輸入採樣：正常值、邊界值（空、超長、0、負數）、不同狀態下（登入/未登入、不同權限、不同時間）。條件行為（「只有月底才…」「只對 VIP…」）是逆向最容易漏的東西，單點觀察必漏。
- **自檢**：每條寫成通則的行為，有 ≥3 個不同輸入的觀察撐著嗎？
- **例**：測一筆訂單發現運費 60 → 寫成「運費固定 60」→ 錯，換一筆超過 1000 的訂單運費是 0——單點觀察把條件邏輯寫成了常數。

## R10 — 先剝掉標準件，只逆向自訂的部分

- **觸發**：對象系統用了已知的框架、library、協定。
- **動作**：先分兩堆：**標準行為**（框架文件寫得清清楚楚的）和**自訂邏輯**（這個系統特有的）。標準堆查文件，不逆向；力氣全部花在自訂堆。判斷法：這段 code 在框架官方範例裡找得到雛形嗎？
- **自檢**：我逆向的東西裡，有沒有其實查文件五分鐘就有答案的部分？
- **例**：花三小時追 Rails 的 session 處理 → 那是框架標準行為，文件都有；該追的是這個 app 塞進 session 的自訂欄位。

## R11 — 交付物是行為規格，不是讀後感

- **觸發**：問題答完，要交付。
- **動作**：產出格式：1. **輸入→輸出規格表**（含條件分支）。2. 狀態圖或流程圖（照 domains/DIAGRAMS.md）。3. **已驗證／推測分開標**（J2 分級）。4. 附 MAP.md 的未解問題清單。禁止交「大致上它是一個處理訂單的系統」式敘述文——下一個人拿你的交付物要能直接用，不是再讀一遍。
- **自檢**：拿我的規格表能預測系統對一筆新輸入的行為嗎？

---

## 回合終檢（結束前逐題答，任一 no → 先修再結束）

1. 有具體問題，且答的是它（不是漫遊）？
2. 每條結論來自行為觀察，不是名字/註解推測？
3. 通則都有 ≥3 種輸入的採樣？
4. 主動實驗全部在副本上？
5. MAP.md 是最新的（含死路）？
6. 交付物是規格表＋已驗證/推測分標？

## 禁語表

| 禁語 | 替代寫法 |
|---|---|
| 「看名字應該是做 ___ 的」 | 「名字暗示 ___，實驗驗證：<做了什麼、看到什麼>」 |
| 「大概看懂了」 | 「已驗證行為 N 條（見規格表），未解 M 條（見清單）」 |
| 「文件說它會 ___」（未實測） | 「文件稱 ___；實測結果 ___」 |
| 「直接在線上試一下」 | 副本上試；線上只觀察 |
| 「這段 code 太亂看不懂」 | 造一筆具體資料追一遍（R4），或 diff 兩個世界（R7） |
