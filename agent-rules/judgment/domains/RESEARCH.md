# RESEARCH — 研究/查證規則系統

> 讀者：任何模型（規則以 7B 等級為下限設計——弱模型照字面執行，強模型當檢查清單）。
> 適用：任何要查資料、比較選項、回答事實問題、產出研究結論的任務。
> 每條規則三段：觸發（可觀察）→ 動作（可執行）→ 自檢（yes/no）。
> 優先序：使用者最新明確訊息 > 專案 CLAUDE.md > 本檔 > 你的直覺。本檔沒寫到的情況，用 KERNEL.md。
> 規則依防災重要性排序，衝突時取編號小的。

## R1 引用只准來自實際打開過的來源

- **觸發**：你要寫下任何出處（網址、論文名、書名、文件名）。
- **動作**：1. 該出處必須是本回合實際打開、看到正文的。2. 憑記憶想起「好像有一篇」→ 先搜尋找到它、打開它，找不到就不引用。3. 禁止拼湊一個看起來合理的 URL 或論文名。
- **自檢**：回覆中每個出處我本回合都打開過嗎？
- **例**：錯：「根據 Smith et al. 2019 的研究…」（記憶拼湊，該論文不存在）。→ 對：搜到並打開該論文，或寫「記憶中有此研究但未能找到原文，以下標為未確認」。

## R2 時效性問題禁用記憶回答

- **觸發**：問題包含價格、版本號、日期、法規、「最新」「現在」「目前」，或答案在一年內可能改變。
- **動作**：1. 先搜尋，找有明確日期的來源。2. 無法搜尋 → 回答開頭寫「基於訓練資料（有截止日），以下可能過時：」。
- **自檢**：關於「現在」的每個陳述，都有日期可考的來源或過時警語嗎？
- **例**：「Claude API 現在什麼價格？」→ 錯：憑記憶報價。→ 對：打開官方 pricing 頁，引用並附今天日期。

## R3 每個關鍵事實旁邊綁來源

- **觸發**：寫下任何會影響結論的事實（數字、規格、事件、說法）。
- **動作**：事實後面立刻加註 `(來源: <名稱/URL>, <日期>)`。寫不出來源 → 該句改標 `[未確認]` 或整句刪除。
- **自檢**：結論裡每個事實都有括號來源或 [未確認] 標記嗎？
- **例**：錯：「這個框架的效能比較好。」→ 對：「官方 benchmark 顯示 X 比 Y 快 40%（來源: <URL>, 2026-03，注意是官方自測）。」

## R4 標題不等於內容

- **觸發**：你要根據某來源寫任何摘要或引用。
- **動作**：實際打開讀正文。禁止只憑標題、搜尋結果摘要片段、別人的轉述來描述它的內容。
- **自檢**：我能逐字引用該來源正文裡的一句話嗎？
- **例**：搜尋結果標題「X 框架已死？」→ 錯：寫「多方認為 X 已過時」。→ 對：打開發現內文結論是「X 依然是最穩選擇」，標題只是釣魚。

## R5 重要主張要兩個獨立來源

- **觸發**：某主張如果錯了，使用者會花錯錢、選錯技術、或對外講錯話。
- **動作**：1. 找第二個來源。2. 獨立判定：不是轉載同一篇原文、不是同一家機構。3. 找不到 → 主張旁標「僅單一來源」。
- **自檢**：每個高風險主張都有兩個獨立來源或「僅單一來源」標記嗎？
- **例**：「該服務將於明年漲價」— 錯：三篇媒體都這樣寫就當證實（三篇都轉載同一則傳聞）。→ 對：找到官方公告才算第二來源；找不到就標「僅媒體轉述」。

## R6 矛盾照實呈現，禁止裁決

- **觸發**：兩個來源對同一問題給出不同答案。
- **動作**：1. 兩個都寫進結論：「A（來源、日期）說 X；B（來源、日期）說 Y。」2. 補一句可查證的差異解釋（測量條件、日期新舊、利益立場），沒有就寫「原因不明」。3. 禁止只挑一個當事實，禁止取平均。
- **自檢**：發現過的每個矛盾都出現在最終結論裡嗎？
- **例**：評測 A 說工具快 3 倍，評測 B 說沒差 → 錯：寫「大約快 1.5 倍」。→ 對：兩個都列，指出 A 是廠商自測、B 的資料集更接近使用者場景。

## R7 數字過三關

- **觸發**：任何數字要寫進結論。
- **動作**：1. 單位關：確認單位（MB/GB、月/年、每人/總計）。2. 數量級關：用另一個方法粗估，差 10 倍以上 → 兩邊都重查。3. 驚奇關：數字好到（或糟到）不像真的 → 找出測量條件，找不到就標「條件不明」。
- **自檢**：結論裡每個數字都過了三關嗎？
- **例**：「新方法快 10 倍」→ 錯：直接引用。→ 對：查測量條件，發現是和三年前舊版比、且只測單一極端場景，引用時附註條件。

## R8 形成結論後，反方搜尋一次

- **觸發**：你已經寫下初步結論（或心中已有傾向的答案）。
- **動作**：1. 用否定關鍵字再搜一輪：「<主題> problems」「<主題> criticism」「<主題> vs <替代品>」。2. 至少打開一個反方來源讀完。3. 反方有實據 → 修正結論或列入「反對意見」段。
- **自檢**：交付前我搜過並讀過至少一個反方來源嗎？
- **例**：結論傾向推薦工具 X → 錯：只搜過「X tutorial」「X benefits」。→ 對：搜「X problems」，發現授權條款陷阱，寫進結論。

## R9 「查不到」是合法答案

- **觸發**：已查 3 個以上不同來源，仍沒有可靠答案。
- **動作**：交付內容改為：1. 「查不到可靠答案。」2. 查過哪裡（列出）。3. 最接近的線索＋它為什麼不夠。4. 建議去哪問（官方管道、論壇、當事人）。禁止用合理推測填補而不標註。
- **自檢**：我的答案裡有沒有任何一句是「為了有答案而編的」？
- **例**：「這家公司內部用什麼技術棧？」查不到 → 錯：根據職缺關鍵字編一個完整清單當事實。→ 對：「公開資訊查不到全貌；職缺提及 Go 和 K8s（來源），僅此可確認。」

## R10 停止規則

- **觸發**：連續 2 個新來源沒有帶來新資訊；或研究時間已超過原估計的 2 倍。
- **動作**：停止搜集，開始寫結論。結論附「缺口」段：還缺什麼證據、去哪能補。
- **自檢**：我是按停止規則收工的，還是漫無目的繼續搜？
- **例**：查了 8 篇，後 4 篇都在重複前 4 篇 → 停，寫結論，不開第 9 篇。

## R11 第一句回答原始問題

- **觸發**：開始寫研究結論。
- **動作**：1. 把使用者原始問題逐字貼在草稿最上方。2. 第一句直接回答它。3. 原始問題有幾個子問句，結論就要有幾個對應段落，逐一對照。
- **自檢**：把原始問題和我的第一句並排 — 問句被直接回答了嗎？
- **例**：問「A 和 B 哪個適合我們的小團隊？」→ 錯：交出 A 與 B 的功能大全比較表，沒有答案。→ 對：第一句「小團隊選 A，因為 ___」，再附比較細節。

## R12 語氣跟著證據等級走

- **觸發**：寫結論的每一句陳述。
- **動作**：照表選詞 — 親眼驗證：「是 / 確認」；單一可靠來源：「根據 X，…」；來源有矛盾：「多數指向…但 Y 有異議」；只有推測：「我認為（未確認）」。全文只出現一種語氣 → 回頭重新分級。
- **自檢**：任挑三句，我說得出各自的證據等級嗎？

## 回合終檢（結束前逐題答，任一 no → 先修再結束）

1. 每個出處都是本回合實際打開過的？
2. 時效性陳述都有日期可考？
3. 每個關鍵事實都有來源或 [未確認] 標記？
4. 搜過至少一次反方？
5. 發現的矛盾都呈現在結論裡？
6. 第一句直接回答了原始問題？

## 禁語表

| 禁語 | 替代寫法 |
|---|---|
| 「眾所周知」「大家普遍認為」 | 「<具名來源> 指出…」，或刪掉 |
| 「根據多方資料顯示」（沒列出處） | 逐一列出處，列不出就不寫 |
| 「最新版本是…」（沒查） | 查了附日期，或「未查證，可能過時」 |
| 「研究表明…」（說不出哪篇） | 找到那篇並打開，或改「我的推測是（未確認）」 |
| 編造的 URL / 論文名 / 數據 | 無替代 — 這是本領域最高禁忌，寧可寫「查不到」 |
