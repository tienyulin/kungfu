# 清單集

## 逼問清單（Step 3，寫每端點 AC 時逐題回答，答案寫進 AC —— 不准留白）

- 每個 **optional 欄位**：None 時行為？
- 每個**比較**：邊界含不含等號？
- 每個**替代輸入形式**（同一資源可用 id / 名稱 / 時間戳等多種方式指定時）：誰負責
  解析成 canonical 形式？用什麼方法？解析失敗回什麼？每個形式都要有自己的 AC
- **dry_run** response 有哪些欄位？回退資訊（prior state）在 dry_run 給不給？
- dry_run 的預測範圍：執行後才知道的值（新 id、新版本號）**不預測**，明寫哪些不給
- 操作**耗時**？sync 或 202？timeout 責任在誰？
- **並發**：兩請求同打一資源會怎樣？（狀態機擋 or 未防護＋風險說明）
- **冪等**：同請求打兩次，第二次回什麼？
- **輔助性 mutation**（執行前自動修復前置條件的副作用，例如先改設定再操作）：
  必須在**其餘全部前置條件確定通過後**才執行 —— 注定失敗的請求不得留下副作用
- 固定文案/token：spec 給字面值＋常數檔名

## 盲審閘門（Step 5）

用 subagent 機制 spawn 一個 subagent（有 read-only 型就用它；隔離是 prompt 層約束，
選型只是加一道保險），prompt **整段照抄**（`<spec 路徑>` 換掉）：

> 你是一個實作前的 spec 盲審員，對本專案零背景。**只准讀這一個檔案：`<spec 路徑>`。
> 禁止讀任何其他檔案（包括 SOP、repo 程式碼、其他 spec），禁止搜尋 repo。**
> 任務：假設你要照這份 spec 實作，列出「實作者必須猜測或發明」的每一處。
> 每條標嚴重度：兩個合理實作者會做出不同行為 = HIGH；要自行決定並記錄 = MED；
> 純外觀 = LOW。引用 spec 原文。嚴格，不要稱讚。
> 最後另起一節回答：只讀 Part A，一個不懂技術的審批者能否說出「這 API 做什麼、
> 最危險的操作是什麼、防護是什麼」？答不出的點列出來。

Part A 那節答不出 = Part A 不合格，與 HIGH 同級處理。

**分流規則**：
- 每條發現**逐字對回 spec 原文**再判定；spec 已明文的 → 駁回並記理由（駁回要引原文）
- 分流時**可改判嚴重度**（reviewer 的 HIGH 實為 MED 就降級），改判要在 REVIEWS.md
  記理由；後續規則以**改判後**的嚴重度為準。「部分成立的 HIGH」若未定義行為仍存在，
  算成立的 HIGH
- 成立的 HIGH > 0 → 修 spec（**連同本輪成立的 MED 一起修**，免得下一輪重複回報）→
  **重新 spawn 全新盲審員**再審一輪，通過前不准開工
- 只有成立的 MED → 修進 spec 才開工，不強制重審
- LOW 選修
- 全部發現與處置記入 **spec 同資料夾**的 `REVIEWS.md`（例 `docs/specs/<組名>/REVIEWS.md`）

### `REVIEWS.md` 格式（沒有就建檔，每輪 append 一節）

```markdown
## <spec-slug> — 盲審第 <n> 輪（<日期>）

| # | 嚴重度 | 發現（引 spec 原文） | 判定 | 處置 |
|---|--------|--------------------|------|------|
| 1 | HIGH  | 「...」§3 未定義 X | 成立 | spec §3 補 AC-FD-7 |
| 2 | MED   | 「...」            | 駁回 | §0 已明文（引原文） |
| 3 | MED   | 「...」            | 部分成立 | 只修 response 範例，行為已定義 |

結果：HIGH 0 / MED 2 / LOW 1 → 通過（或：修 spec 後進第 <n+1> 輪）
```

## 實作回饋歸因表（Step 6）

| 歸因 | 動作 |
|------|------|
| SOP 缺資訊（錯誤碼缺漏等） | 補 SOP ＋ spec |
| skill 模板/逼問清單沒問到 | 改 skill ＋ 重產 spec 該節 |
| spec 產出沒照模板 | 重產 spec 該節 |
| 純 code bug | 修 code ＋ 補 AC 對應測試 |

修完重跑 Step 4；重大修改重跑 Step 5。全程記 spec 同資料夾的 `REVIEWS.md`（歸因也 append 成一節）。
